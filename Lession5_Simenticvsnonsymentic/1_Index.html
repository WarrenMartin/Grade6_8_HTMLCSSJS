Semantic vs Non-Semantic HTML Tags

Page 0: Semantic vs Non-Semantic HTML Tags

Semantic HTML tags are tags that carry meaning and convey the structure of a web page. They provide context to the content. Some common semantic tags include:

  • <header> - Represents a container for introductory content or a set of navigational links.
  • <nav> - Defines a section of navigation links.
  • <article> - Represents self-contained content, such as a news article or a blog post.
  • <section> - Represents a thematic grouping of content.

Non-Semantic HTML Tags

Non-semantic HTML tags are tags used for layout and presentation, without conveying the specific meaning or purpose of the enclosed content. Examples of non-semantic tags include:

  • <div> - A generic container for grouping and styling content.
  • <span> - A generic inline container for styling or scripting.
  • <table> - Used for creating tabular data, but without specifying the nature of the content.
  • <b> and <i> - Used for bold and italic text, respectively, but they don't convey the meaning of the text.
